# Chronos 复现实验操作指南

本文档不是 README，用于记录本项目的实际执行方式。当前推荐目标是：在 V100 单卡约 10 小时的限制下，尽量得到一组可汇报的 Chronos 轻量复现结果。

## 1. 复现目标

完整 Chronos 论文训练使用 10M TSMixup + 1M KernelSynth、200K steps、有效 batch size 256，并在多数据集 benchmark 上比较 WQL / MASE。V100 单卡 10 小时不足以完整复现 Small 或完整 200K 训练。

本项目采用更现实的目标，并把 Tiny 和 Mini 分成两条脚本：

```text
主实验 Tiny：Chronos-T5-Tiny + 官方训练语料 v100 子集 + 50K steps
辅助实验 Mini：Chronos-T5-Mini + 官方训练语料 v100 子集 + 10K steps，视剩余时间执行
对照实验：Seasonal Naive baseline + 官方 Chronos checkpoint zero-shot
```

这样可以验证 Chronos 的核心训练范式、数据混合方式、checkpoint/断点能力，以及训练后相对基线的 forecasting 表现。

## 2. 数据策略

主线数据来自 HuggingFace 官方 Chronos 训练语料：

```text
dataset_name: autogluon/chronos_datasets

TSMixup:
  training_corpus_tsmixup_10m

KernelSynth:
  training_corpus_kernel_synth_1m
```

混合比例保持论文设定：

```text
TSMixup : KernelSynth = 9 : 1
```

训练运行时会显式记录：

```text
dataset_name
data_level
training_data_paths
probability
```

记录位置：

```text
training_preflight.json
training_progress.json
checkpoint-final/training_info.json
```

## 3. 数据级别

| data_level | TSMixup | KernelSynth | Total | 用途 |
|---|---:|---:|---:|---|
| debug | 900 | 100 | 1,000 | 测试下载、转换、训练链路 |
| conservative | 45,000 | 5,000 | 50,000 | 网络慢或预算紧张 |
| recommended | 90,000 | 10,000 | 100,000 | Tiny 主实验推荐 |
| stretch | 180,000 | 20,000 | 200,000 | 时间和网络都顺利时 |
| v100 | 900,000 | 100,000 | 1,000,000 | V100 10 小时主实验 |

说明：`v100` 不是论文 full corpus，但保持官方数据源和 9:1 比例，规模是 `recommended` 的 10 倍，适合 V100 单卡时间限制。

> ⚠️ **HF streaming 行为**：第一条数据下载需要 ~60s（拉取整个 parquet 分片），后续速度可达 100+ 条/秒。不要因最初的等待中断脚本。
> conservative 级别实测：TSMixup 54,187 seen → 45,000 written（过滤了 ~17% 太短/nan过多的序列），约 5 分钟完成。

## 3. 环境准备

```bash
python --version
pip install --upgrade pip
pip install -e ".[dev,test]"
```

如果需要从 `checkpoint-xxxxx` 完整续训，云端 torch 版本建议为 `>=2.6`。较新的 Transformers 会因为 `torch.load` 安全限制，拒绝在 torch `<2.6` 时加载 `optimizer.pt`。检查方式：

```bash
python -c "import torch; print(torch.__version__)"
```

如果版本低于 2.6，优先在云端环境中升级 torch，或者退而求其次使用“只从模型权重继续训练”：

```bash
python scripts/training/train.py \
  --config scripts/training/configs/chronos_t5_tiny_v100.yaml \
  --model-id outputs/checkpoints/chronos_t5_tiny_v100_50k/run-0/checkpoint-25000 \
  --no-random-init \
  --output-dir ./outputs/checkpoints/chronos_t5_tiny_v100_continue_from_weights
```

两者区别：

```text
resume_from_checkpoint:
  恢复模型、optimizer、scheduler、step，适合真正断点续训

model_id + no-random-init:
  只从模型权重继续，optimizer 和 scheduler 重新开始，适合作为保底方案
```

建议设置 HuggingFace 缓存，避免重复下载：

```bash
export HF_HOME=/home/ma-user/work/cache/huggingface
export HF_DATASETS_CACHE=/home/ma-user/work/cache/huggingface/datasets
export TRANSFORMERS_CACHE=/home/ma-user/work/cache/huggingface/transformers
export TOKENIZERS_PARALLELISM=false
```

确认 GPU：

```bash
nvidia-smi
python -c "import torch; print(torch.cuda.is_available()); print(torch.cuda.get_device_name(0))"
```

## 4. 准备官方 Corpus 子集

网络慢时优先用 **conservative**（50k 条，~5 分钟）：

```bash
python scripts/data/prepare_chronos_corpus_subset.py \
  --output_dir data/processed \
  --data_level conservative \
  --min_length 288 \
  --max_length 2048 \
  --seed 42 \
  --compression lz4
```

Debug 数据（1k 条，测链路）：

```bash
python scripts/data/prepare_chronos_corpus_subset.py \
  --output_dir data/processed \
  --data_level debug \
  --min_length 288 \
  --max_length 1024 \
  --seed 42 \
  --compression lz4 \
  --overwrite
```

推荐/极限数据（网络充裕时）：

```bash
# recommended: 100k
python scripts/data/prepare_chronos_corpus_subset.py \
  --output_dir data/processed \
  --data_level v100 \
  --min_length 576 \
  --max_length 2048 \
  --seed 42 \
  --compression lz4
```

输出文件：

```text
data/processed/chronos_debug_tsmixup_900.arrow
data/processed/chronos_debug_kernel_synth_100.arrow

data/processed/chronos_tsmixup_45k.arrow        # conservative
data/processed/chronos_kernel_synth_5k.arrow

data/processed/chronos_tsmixup_90k.arrow        # recommended
data/processed/chronos_kernel_synth_10k.arrow

data/processed/chronos_corpus_subset_summary.json
data/processed/chronos_corpus_subset_progress.json
```

检查数据：

```bash
python scripts/data/inspect_arrow_dataset.py \
  --paths data/processed/chronos_tsmixup_900k.arrow data/processed/chronos_kernel_synth_100k.arrow \
  --freq h \
  --min_length 576 \
  --max_examples 5
```

`chronos_corpus_subset_summary.json` 必须保留，用于报告数据来源、数量、过滤统计和 sha256。

数据准备脚本是流式写入 Arrow 文件，不会把 100 万条序列一次性攒入内存。如果中途失败，临时文件会删除；已完成的 TSMixup 或 KernelSynth 文件会在下次运行时跳过，除非显式传入 `--overwrite`。

## 8. V100 Tiny 主实验

配置文件：

```text
scripts/training/configs/chronos_t5_tiny_v100.yaml
```

核心参数：

```text
model_id: google/t5-efficient-tiny
data_level: v100
context_length: 512
prediction_length: 64
max_steps: 50000
per_device_train_batch_size: 16
gradient_accumulation_steps: 8
effective_batch_size: 128
fp16: true
tf32: false
torch_compile: false
```

运行：

```bash
bash scripts/cloud/tiny_v100_train.sh
```

如果前 1000 steps 很快，并且剩余时间充足，可以覆盖步数继续跑：

```bash
python scripts/training/train.py \
  --config scripts/training/configs/chronos_t5_tiny_v100.yaml \
  --max-steps 100000 \
  --save-steps 10000 \
  --output-dir ./outputs/checkpoints/chronos_t5_tiny_v100_100k
```

如果云端中断，找到最近 checkpoint 后续跑：

```bash
export RESUME_FROM_CHECKPOINT=outputs/checkpoints/chronos_t5_tiny_v100_50k/run-0/checkpoint-25000
bash scripts/cloud/tiny_v100_train.sh
```

注意：续跑会继续写入当前配置指定的输出目录；报告中必须记录续跑来源 checkpoint。

```bash
python scripts/data/generate_synthetic_fallback.py \
  --output_path data/processed/synthetic_fallback_1k.arrow \

  --config scripts/training/configs/chronos_t5_tiny_debug.yaml
```

通过标准：

```text

4. 不出现 OOM

6. training_progress.json 最终 stage 为 complete
```

## 9. Tiny 正式训练

### 路线 A：真实数据（推荐）

用 conservative（50k 真实 corpus，已验证通过）：

```bash

  --config scripts/training/configs/chronos_t5_tiny_real.yaml
```

```bash
  --num_series 50000 --seed 42 --data_level fallback

  --config scripts/training/configs/chronos_t5_tiny_fallback.yaml
```

## 11. 本地评估与后处理

## 12. OOM / 加速调整

T5-Tiny 在 T4 上非常轻量（batch=32 时显存仅 ~1.5G），一般不会 OOM。

加速策略：

```text
# T4 上 Tiny 最佳配置（已验证）
per_device_train_batch_size: 32    # 越大越快，Tiny 撑得住
gradient_accumulation_steps: 1     # 无需累积，一步到位

# 如果 batch=64 OOM，回退：
per_device_train_batch_size: 8
gradient_accumulation_steps: 4
```

如果换更大模型 OOM，按顺序降低：

```text
per_device_train_batch_size: 32 -> 16 -> 8 -> 4
context_length: 256 -> 128
prediction_length: 32 -> 16
```

## 13. 快速评估 (MASE/WQL)

训练完成后，用本地 quick_eval 评估：

```bash
# 评估 200 条（~30 秒）
python scripts/evaluation/quick_eval.py \
  --checkpoint outputs/checkpoints/chronos_t5_tiny_real/run-0/checkpoint-final \
  --n_series 200
```

输出示例：

```text
========================================
  Series evaluated: 200
  MASE = 1.8821
  WQL  = 0.1702
========================================
```

> MASE < 1 表示优于 naive 预测（论文 fully trained Small 模型约 0.8-1.0）。
> Tiny + 5k 步 + 合成数据 MASE≈1.9 属正常预期，换真实数据 + 多步数后会明显改善。

## 14. 推荐执行顺序

```bash
# 0. 环境变量
source setup_env.sh

# 1. 安装环境
pip install -e ".[dev,test]"

# 2. Debug 训练（测链路，~14 秒）
python scripts/data/generate_synthetic_fallback.py \
  --output_path data/processed/synthetic_fallback_1k.arrow \
  --num_series 1000 --seed 42 --data_level debug
python scripts/training/train.py \
  --config scripts/training/configs/chronos_t5_tiny_debug.yaml

# 3. 准备真实数据（conservative 50k，~5 分钟）
python scripts/data/prepare_chronos_corpus_subset.py \
  --data_level conservative --output_dir data/processed

# 4. 正式训练（5000 步，~35 分钟，T4）
python scripts/training/train.py \
  --config scripts/training/configs/chronos_t5_tiny_real.yaml

# 5. 快速评估 MASE/WQL
python scripts/evaluation/quick_eval.py \
  --checkpoint outputs/checkpoints/chronos_t5_tiny_real/run-0/checkpoint-final \
  --n_series 200
```

不要提交 `data/` 和 `outputs/` 到 Git；它们已在 `.gitignore` 中忽略。
