# Chronos 轻量复现操作指南

本文档说明如何使用本仓库完成 Chronos-T5-Tiny 轻量训练复现、数据准备、云端训练和故障排查。数据后处理、指标统计和画图建议在本地执行；云端主要负责数据生成和模型训练。

## 1. 环境准备

推荐环境：

```bash
python --version  # 建议 3.10 或 3.11
pip install --upgrade pip
pip install -e ".[dev,test]"
```

确认 GPU：

```bash
nvidia-smi
python -c "import torch; print(torch.cuda.is_available()); print(torch.cuda.get_device_name(0))"
```

## 2. 准备数据

生成可断点恢复的数据：

```bash
python scripts/prepare_data.py \
  --output_dir data/processed \
  --num_series 2000 \
  --min_length 384 \
  --max_length 1024 \
  --context_length 256 \
  --prediction_length 32 \
  --seed 42
```

如果中断，重新执行同一命令即可。脚本会复用已经写入的 `raw_series.jsonl`，不会从零开始浪费时间。

生成完成后检查：

```bash
cat data/processed/data_manifest.json
cat data/processed/prepare_progress.json
```

关键输出：

```text
data/processed/train.arrow
data/processed/validation.jsonl
data/processed/test.jsonl
data/processed/data_manifest.json
data/processed/data_quality.csv
```

`data_manifest.json` 中包含文件大小、sha256、样本数量和跳过数量，用来确认数据完整性。

## 3. 云端 Debug 训练

正式训练前先跑 debug 配置：

```bash
python scripts/training/train.py \
  --config scripts/training/configs/chronos_t5_tiny_debug.yaml
```

通过标准：

```text
1. 训练能启动
2. loss 正常输出
3. 不出现 NaN
4. 不出现 OOM
5. checkpoint-final 能保存
6. training_progress.json 最终 stage 为 complete
```

查看诊断文件：

```bash
cat outputs/checkpoints/debug_tiny/run-*/training_progress.json
cat outputs/checkpoints/debug_tiny/run-*/training_preflight.json
```

如果失败：

```bash
cat outputs/checkpoints/debug_tiny/run-*/training_failure.json
```

## 4. Tiny 正式训练

Debug 通过后运行 Tiny 主实验：

```bash
python scripts/training/train.py \
  --config scripts/training/configs/chronos_t5_tiny_t4.yaml
```

默认配置：

```text
context_length = 256
prediction_length = 32
max_steps = 5000
save_steps = 1000
batch_size = 8
gradient_accumulation_steps = 8
torch_compile = false
tf32 = false
```

输出目录：

```text
outputs/checkpoints/chronos_t5_tiny_t4/run-*/
```

重点检查：

```bash
cat outputs/checkpoints/chronos_t5_tiny_t4/run-*/training_progress.json
ls outputs/checkpoints/chronos_t5_tiny_t4/run-*/checkpoint-final
```

## 5. 日志和故障排查

训练脚本会自动记录以下文件：

```text
training_preflight.json
training_progress.json
training_failure.json
```

用途：

```text
training_preflight.json  记录配置、环境、GPU、数据路径和文件大小
training_progress.json   记录当前训练阶段
training_failure.json    失败时记录异常类型、异常信息和 traceback
```

常见阶段：

```text
config_parsed
data_preflight_complete
datasets_constructed
model_initializing
model_initialized
chronos_dataset_ready
training_args_initializing
trainer_initializing
trainer_initialized
training_started
training_finished
saving_final_checkpoint
complete
failed
```

如果云端失败，优先看：

```bash
cat outputs/checkpoints/*/run-*/training_progress.json
cat outputs/checkpoints/*/run-*/training_failure.json
cat outputs/checkpoints/*/run-*/training_preflight.json
```

这样可以快速判断是数据路径、模型下载、Trainer 初始化、训练过程还是 checkpoint 保存出错。

## 6. 本地 Smoke 测试

本地快速验证训练链路：

```bash
python scripts/training/train.py \
  --config scripts/training/configs/local-gpu-smoke.yaml
```

本地稍重一点的测试：

```bash
python scripts/training/train.py \
  --config scripts/training/configs/local-gpu-t5-debug.yaml
```

这两个配置只用于本地排查，不作为正式复现实验结果。

## 8. 推荐执行顺序

```bash
# 1. 安装环境
pip install -e ".[dev,test]"

# 2. 准备数据
python scripts/prepare_data.py --output_dir data/processed

# 3. Debug 训练
python scripts/training/train.py --config scripts/training/configs/chronos_t5_tiny_debug.yaml

# 4. Tiny 正式训练
python scripts/training/train.py --config scripts/training/configs/chronos_t5_tiny_t4.yaml

# 5. 检查结果
cat outputs/checkpoints/chronos_t5_tiny_t4/run-*/training_progress.json
```

## 9. 注意事项

如果 HuggingFace 模型下载失败，可以先在网络较好的环境缓存 `google/t5-efficient-tiny`，或改用本地模型配置做 smoke 测试。

如果 T4 上 OOM，按顺序降低：

```text
per_device_train_batch_size: 8 -> 4 -> 2
context_length: 256 -> 128
prediction_length: 32 -> 16
gradient_accumulation_steps: 8 -> 16 -> 32
```

不要把 `data/` 和 `outputs/` 提交到 Git；它们已被 `.gitignore` 忽略。
