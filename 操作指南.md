# Chronos 轻量复现操作指南

本文档说明如何准备数据、在云端训练 Chronos-T5-Tiny，以及如何排查故障。本文档不是 README，作为实验执行说明使用。

## 1. 数据策略

本项目主线数据不再使用单一 toy dataset，而是优先使用 Chronos 官方 HuggingFace 数据集的训练语料子集：

```text
dataset_name: autogluon/chronos_datasets

TSMixup:
  training_corpus_tsmixup_10m

KernelSynth:
  training_corpus_kernel_synth_1m
```

混合比例尽量贴近论文：

```text
TSMixup : KernelSynth = 9 : 1
```

训练运行时会显式记录：

```text
dataset_name
data_level
training_data_paths
probability
```

这些信息会写入：

```text
training_preflight.json
training_progress.json
checkpoint-final/training_info.json
```

## 2. 数据级别

支持四个数据级别：

| data_level | TSMixup | KernelSynth | Total | 用途 |
|---|---:|---:|---:|---|
| debug | 900 | 100 | 1,000 | 测试下载、转换、训练链路 |
| conservative | 45,000 | 5,000 | 50,000 | 网络慢或预算紧张 |
| recommended | 90,000 | 10,000 | 100,000 | Tiny 主实验推荐 |
| stretch | 180,000 | 20,000 | 200,000 | 时间和网络都顺利时 |

## 3. 环境准备

```bash
python --version
pip install --upgrade pip
pip install -e ".[dev,test]"
```

建议设置 HuggingFace 缓存：

```bash
export HF_HOME=/home/ma-user/work/cache/huggingface
export HF_DATASETS_CACHE=/home/ma-user/work/cache/huggingface/datasets
export TRANSFORMERS_CACHE=/home/ma-user/work/cache/huggingface/transformers
export TOKENIZERS_PARALLELISM=false
```

确认 GPU：

```bash
nvidia-smi
python -c "import torch; print(torch.cuda.is_available()); print(torch.cuda.get_device_name(0))"
```

## 4. 准备官方 Corpus 子集

Debug 数据：

```bash
python scripts/data/prepare_chronos_corpus_subset.py \
  --output_dir data/processed \
  --data_level debug \
  --min_length 288 \
  --max_length 1024 \
  --seed 42 \
  --compression lz4 \
  --overwrite
```

主线推荐数据：

```bash
python scripts/data/prepare_chronos_corpus_subset.py \
  --output_dir data/processed \
  --data_level recommended \
  --min_length 288 \
  --max_length 2048 \
  --seed 42 \
  --compression lz4
```

输出示例：

```text
data/processed/chronos_debug_tsmixup_900.arrow
data/processed/chronos_debug_kernel_synth_100.arrow

data/processed/chronos_tsmixup_90k.arrow
data/processed/chronos_kernel_synth_10k.arrow

data/processed/chronos_corpus_subset_summary.json
data/processed/chronos_corpus_subset_progress.json
```

`chronos_corpus_subset_summary.json` 会记录：

```text
dataset_name
data_level
tsmixup_config
kernel_config
requested / written
skipped statistics
output_files
sha256
```

## 5. 检查 Arrow 数据

Debug 数据检查：

```bash
python scripts/data/inspect_arrow_dataset.py \
  --paths data/processed/chronos_debug_tsmixup_900.arrow data/processed/chronos_debug_kernel_synth_100.arrow \
  --freq h \
  --min_length 288 \
  --max_examples 5
```

Recommended 数据检查：

```bash
python scripts/data/inspect_arrow_dataset.py \
  --paths data/processed/chronos_tsmixup_90k.arrow data/processed/chronos_kernel_synth_10k.arrow \
  --freq h \
  --min_length 288 \
  --max_examples 5
```

检查结果保存到：

```text
outputs/logs/inspect_*.json
outputs/logs/inspect_arrow_summary.json
```

## 6. Synthetic Fallback

如果 HuggingFace 下载失败或 streaming 不稳定，使用 synthetic fallback 保底：

```bash
python scripts/data/generate_synthetic_fallback.py \
  --output_path data/processed/synthetic_fallback_50k.arrow \
  --num_series 50000 \
  --min_length 512 \
  --max_length 2048 \
  --seed 42 \
  --data_level fallback
```

如果使用 fallback，报告中必须说明：

```text
由于网络或云端环境限制，官方 Chronos training corpus 子集未能完整采样，
因此使用 synthetic fallback 数据完成训练流程复现。
```

## 8. 云端 Debug 训练

Debug 配置显式使用：

```text
dataset_name: autogluon/chronos_datasets
data_level: debug
training_data_paths:
  - data/processed/chronos_debug_tsmixup_900.arrow
  - data/processed/chronos_debug_kernel_synth_100.arrow
probability:
  - 0.9
  - 0.1
```

运行：

```bash
python scripts/training/train.py \
  --config scripts/training/configs/chronos_t5_tiny_debug.yaml
```

通过标准：

```text
1. 训练能启动
2. loss 正常输出
3. 不出现 NaN
4. 不出现 OOM
5. checkpoint-final 能保存
6. training_progress.json 最终 stage 为 complete
```

## 10. 日志与故障排查

训练脚本会生成：

```text
training_preflight.json
training_progress.json
training_failure.json
```

用途：

```text
training_preflight.json  记录配置、环境、GPU、数据路径和文件大小
training_progress.json   记录当前训练阶段
training_failure.json    失败时记录异常类型、异常信息和 traceback
```

常见阶段：

```text
config_parsed
data_preflight_complete
datasets_constructed
model_initializing
model_initialized
chronos_dataset_ready
training_args_initializing
trainer_initializing
trainer_initialized
training_started
training_finished
saving_final_checkpoint
complete
failed
```

云端失败时优先看：

```bash
cat outputs/checkpoints/*/run-*/training_progress.json
cat outputs/checkpoints/*/run-*/training_failure.json
cat outputs/checkpoints/*/run-*/training_preflight.json
```

## 12. OOM 调整顺序

如果 T4 上 OOM，按顺序降低：

```text
per_device_train_batch_size: 8 -> 4 -> 2
context_length: 256 -> 128
prediction_length: 32 -> 16
gradient_accumulation_steps: 8 -> 16 -> 32
dataloader_num_workers: 2 -> 1
```

## 13. 推荐执行顺序

```bash
# 1. 安装环境
pip install -e ".[dev,test]"

# 2. 准备 debug 数据
python scripts/data/prepare_chronos_corpus_subset.py --data_level debug --output_dir data/processed --overwrite

# 3. 检查 debug 数据
python scripts/data/inspect_arrow_dataset.py \
  --paths data/processed/chronos_debug_tsmixup_900.arrow data/processed/chronos_debug_kernel_synth_100.arrow

# 4. Debug 训练
python scripts/training/train.py --config scripts/training/configs/chronos_t5_tiny_debug.yaml

# 5. 准备 recommended 数据
python scripts/data/prepare_chronos_corpus_subset.py --data_level recommended --output_dir data/processed

# 6. Tiny 正式训练
python scripts/training/train.py --config scripts/training/configs/chronos_t5_tiny_t4.yaml

# 7. 检查结果
cat outputs/checkpoints/chronos_t5_tiny_t4/run-*/training_progress.json
```

不要把 `data/` 和 `outputs/` 提交到 Git；它们已被 `.gitignore` 忽略。
