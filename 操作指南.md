# Chronos 复现实验操作指南

本文档不是 README，用于记录本项目的实际执行方式。当前推荐目标是：在 V100 单卡约 10 小时的限制下，尽量得到一组可汇报的 Chronos 轻量复现结果。

## 1. 复现目标

完整 Chronos 论文训练使用 10M TSMixup + 1M KernelSynth、200K steps、有效 batch size 256，并在多数据集 benchmark 上比较 WQL / MASE。V100 单卡 10 小时不足以完整复现 Small 或完整 200K 训练。

本项目采用更现实的目标：

```text
主实验：Chronos-T5-Tiny + 官方训练语料 v100 子集 + 50K steps
辅助实验：Chronos-T5-Mini + 官方训练语料 v100 子集 + 10K steps
对照实验：Seasonal Naive baseline + 官方 Chronos checkpoint zero-shot
```

这样可以验证 Chronos 的核心训练范式、数据混合方式、checkpoint/断点能力，以及训练后相对基线的 forecasting 表现。

## 2. 数据策略

主线数据来自 HuggingFace 官方 Chronos 训练语料：

```text
dataset_name: autogluon/chronos_datasets

TSMixup:
  training_corpus_tsmixup_10m

KernelSynth:
  training_corpus_kernel_synth_1m
```

混合比例保持论文设定：

```text
TSMixup : KernelSynth = 9 : 1
```

训练运行时会显式记录：

```text
dataset_name
data_level
training_data_paths
probability
```

记录位置：

```text
training_preflight.json
training_progress.json
checkpoint-final/training_info.json
```

## 3. 数据级别

| data_level | TSMixup | KernelSynth | Total | 用途 |
|---|---:|---:|---:|---|
| debug | 900 | 100 | 1,000 | 检查下载、转换、训练链路 |
| conservative | 45,000 | 5,000 | 50,000 | 网络或预算紧张时使用 |
| recommended | 90,000 | 10,000 | 100,000 | T4 / 本地较小实验 |
| stretch | 180,000 | 20,000 | 200,000 | T4 较完整实验 |
| v100 | 900,000 | 100,000 | 1,000,000 | V100 10 小时主实验 |

说明：`v100` 不是论文 full corpus，但保持官方数据源和 9:1 比例，规模是 `recommended` 的 10 倍，适合 V100 单卡时间限制。

## 4. 环境准备

```bash
python --version
pip install --upgrade pip
pip install -e ".[dev,test]"
```

建议设置 HuggingFace 缓存，避免重复下载：

```bash
export HF_HOME=/home/ma-user/work/cache/huggingface
export HF_DATASETS_CACHE=/home/ma-user/work/cache/huggingface/datasets
export TRANSFORMERS_CACHE=/home/ma-user/work/cache/huggingface/transformers
export TOKENIZERS_PARALLELISM=false
```

确认 GPU：

```bash
nvidia-smi
python -c "import torch; print(torch.cuda.is_available()); print(torch.cuda.get_device_name(0))"
```

## 5. Debug 数据与训练

先准备 debug 数据：

```bash
python scripts/data/prepare_chronos_corpus_subset.py \
  --output_dir data/processed \
  --data_level debug \
  --min_length 288 \
  --max_length 1024 \
  --seed 42 \
  --compression lz4 \
  --overwrite
```

检查 Arrow 数据：

```bash
python scripts/data/inspect_arrow_dataset.py \
  --paths data/processed/chronos_debug_tsmixup_900.arrow data/processed/chronos_debug_kernel_synth_100.arrow \
  --freq h \
  --min_length 288 \
  --max_examples 5
```

运行 debug 训练：

```bash
python scripts/training/train.py \
  --config scripts/training/configs/chronos_t5_tiny_debug.yaml
```

通过标准：

```text
训练能启动
loss 正常输出
无 NaN
无 OOM
checkpoint-final 能保存
training_progress.json 最终 stage 为 complete
```

## 6. V100 数据准备

V100 主实验数据：

```bash
python scripts/data/prepare_chronos_corpus_subset.py \
  --output_dir data/processed \
  --data_level v100 \
  --min_length 576 \
  --max_length 2048 \
  --seed 42 \
  --compression lz4
```

输出文件：

```text
data/processed/chronos_tsmixup_900k.arrow
data/processed/chronos_kernel_synth_100k.arrow
data/processed/chronos_corpus_subset_summary.json
data/processed/chronos_corpus_subset_progress.json
```

检查数据：

```bash
python scripts/data/inspect_arrow_dataset.py \
  --paths data/processed/chronos_tsmixup_900k.arrow data/processed/chronos_kernel_synth_100k.arrow \
  --freq h \
  --min_length 576 \
  --max_examples 5
```

`chronos_corpus_subset_summary.json` 必须保留，用于报告数据来源、数量、过滤统计和 sha256。

## 7. V100 Tiny 主实验

配置文件：

```text
scripts/training/configs/chronos_t5_tiny_v100.yaml
```

核心参数：

```text
model_id: google/t5-efficient-tiny
data_level: v100
context_length: 512
prediction_length: 64
max_steps: 50000
per_device_train_batch_size: 16
gradient_accumulation_steps: 8
effective_batch_size: 128
fp16: true
tf32: false
torch_compile: false
```

运行：

```bash
python scripts/training/train.py \
  --config scripts/training/configs/chronos_t5_tiny_v100.yaml
```

如果前 1000 steps 很快，并且剩余时间充足，可以覆盖步数继续跑：

```bash
python scripts/training/train.py \
  --config scripts/training/configs/chronos_t5_tiny_v100.yaml \
  --max-steps 100000 \
  --save-steps 10000 \
  --output-dir ./outputs/checkpoints/chronos_t5_tiny_v100_100k
```

## 9. 日志与故障排查

训练脚本会生成：

```text
training_preflight.json
training_progress.json
training_failure.json
```

用途：

```text
training_preflight.json  记录配置、环境、GPU、数据路径和文件大小
training_progress.json   记录当前训练阶段
training_failure.json    失败时记录异常类型、异常信息和 traceback
```

云端失败时优先查看：

```bash
cat outputs/checkpoints/*/run-*/training_progress.json
cat outputs/checkpoints/*/run-*/training_failure.json
cat outputs/checkpoints/*/run-*/training_preflight.json
```

常见处理：

```text
OOM:
  per_device_train_batch_size: 16 -> 8 -> 4
  gradient_accumulation_steps: 8 -> 16 -> 32
  gradient_checkpointing: false -> true

速度太慢:
  先保留 Tiny 50K
  Mini 只跑 5K-10K

数据下载慢:
  保留 chronos_corpus_subset_progress.json
  确认 HF_HOME / HF_DATASETS_CACHE 在持久化目录
```

## 10. 推荐执行顺序

```bash
# 1. 安装环境
pip install -e ".[dev,test]"

# 2. 准备 debug 数据
python scripts/data/prepare_chronos_corpus_subset.py --data_level debug --output_dir data/processed --overwrite

# 3. 检查 debug 数据
python scripts/data/inspect_arrow_dataset.py \
  --paths data/processed/chronos_debug_tsmixup_900.arrow data/processed/chronos_debug_kernel_synth_100.arrow

# 4. Debug 训练
python scripts/training/train.py --config scripts/training/configs/chronos_t5_tiny_debug.yaml

# 5. 准备 V100 数据
python scripts/data/prepare_chronos_corpus_subset.py \
  --data_level v100 \
  --output_dir data/processed \
  --min_length 576 \
  --max_length 2048 \
  --seed 42 \
  --compression lz4

# 6. 检查 V100 数据
python scripts/data/inspect_arrow_dataset.py \
  --paths data/processed/chronos_tsmixup_900k.arrow data/processed/chronos_kernel_synth_100k.arrow \
  --freq h \
  --min_length 576

# 7. Tiny 主实验
python scripts/training/train.py --config scripts/training/configs/chronos_t5_tiny_v100.yaml

# 8. Mini 探针实验
python scripts/training/train.py --config scripts/training/configs/chronos_t5_mini_v100_probe.yaml
```

不要提交 `data/` 和 `outputs/` 到 Git；它们已在 `.gitignore` 中忽略。

## 11. 报告中应如何表述

推荐表述：

```text
在 V100 单卡约 10 小时预算下，本实验使用 Chronos 官方训练语料的 1M 子集，保持 TSMixup:KernelSynth = 9:1、context length 512、prediction length 64，训练 Chronos-T5-Tiny 50K steps，并以 Mini 10K steps 作为模型规模探针。该实验不等同于论文 11M corpus + 200K steps 的完整复现，但可以验证 Chronos 训练范式、数据混合策略和轻量模型在 zero-shot forecasting 上相对基线的趋势。
```
